## 1.【重要,優先處理】貼紙圖層顯示 bug

### 1.1 問題現象
實測發現,裝飾模式**編輯當下**,貼紙(笑臉、蝴蝶結、禮物盒等)可以正常疊在
照片上方顯示;但點擊「完成裝飾」離開編輯模式,回到一般瀏覽畫面後,
**這些貼紙大部分消失不見了,像是被照片或留言卡片蓋住**。手機版封面的星星
貼紙也觀察到類似現象,貼紙疑似被顯示在照片的下層而非上層。

### 1.2 修復要求
請檢查裝飾層(`DecorationLayer.jsx`,第六輪建立的唯讀渲染元件)的
`z-index` 設定,確保**無論是編輯模式或一般瀏覽模式,裝飾貼紙的圖層順序
都必須在照片、留言卡片內容之上**,不能出現「編輯當下看得到、儲存離開
編輯模式後又被蓋住」的落差。這極可能是編輯模式的互動層
(`InteractiveDecorationLayer.jsx`)跟唯讀渲染層各自设定了不同的
`z-index`,導致行為不一致,請一併檢查兩者是否使用一致的圖層層級設定。

### 1.3 驗證要求
- 裝飾完成、離開編輯模式後,重新整理頁面,確認所有貼紙都清楚顯示在
  照片與留言卡片之上,沒有任何貼紙被蓋住或消失
- 手機/平板/桌機三種寬度都要確認,尤其手機版先前有觀察到貼紙位置貼近
  照片邊緣被裁切/蓋住的現象

## 2. 擴大修復「祝福」類別顏色顯示異常

第九輪已修復建卡精靈「下一步」按鈕在「祝福」類別下對比度不足的問題,
但實測發現**同一頁面的步驟指示圓點(頁面上方「1 2 3 4」的當前步驟標示)**
在選擇「祝福」類別時,同樣沒有顯示該有的顏色填滿效果,只有其他三個類別
(生日/紀念日/感謝)正常。

請將第九輪的修復範圍擴大,**全面檢查所有跟著類別強調色(accent color)
變化的畫面元件**(不限於「下一步」按鈕),確保任何使用到類別顏色的地方,
在四種類別底下都不會出現對比度不足、看起來像沒反應/沒作用的情形。
建議做法:延續第九輪建立的 `--on-accent` 自動選字色機制,檢查專案中
是否還有其他元件直接套用 `accent` 顏色但沒有套用這組對比色邏輯,
一併修正,而不是每次發現一個元件才修一個。

## 3.「我的卡片」列表新增刪除功能

### 3.1 需求
「我的卡片」列表頁面,每張卡片新增一個「刪除卡片」的操作選項,
讓使用者可以自行清除測試用或建錯的卡片,不需要手動操作資料庫。

### 3.2 安全性與操作方式
- 刪除是不可逆的破壞性操作,**必須有二次確認**(例如彈出確認視窗,
  說明「此操作無法復原,確定要刪除這張卡片與所有留言嗎?」),避免使用者
  手滑誤刪
- 因為「我的卡片」列表本身就是登入後才看得到(第三輪已建立帳號系統,
  只有卡片建立者本人登入後能看到自己的卡片清單),刪除動作可以直接透過
  使用者當下的登入身份(`auth.uid() = creator_id`)做權限判斷,不需要
  額外走 `admin_key` 驗證的那一套機制
- 資料庫刪除時,依既有的外鍵設定(`ecard_messages` 對 `ecard_cards` 的
  `ON DELETE CASCADE`),刪除卡片時底下所有留言會自動一併清除,
  不需要額外處理

### 3.3 UI 呈現
在「我的卡片」列表每張卡片的「查看」「複製訪客連結」按鈕旁,
新增一個「刪除」按鈕(建議用較不顯眼的顏色或放在較不容易誤觸的位置,
搭配二次確認,降低誤刪風險)。
