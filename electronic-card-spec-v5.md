# 電子卡片專案 — 第五輪修改規格書

> 此文件為前四份規格書(v1 ~ v4)的補充修正,請一併參考。
> 本輪處理兩件事:① 留言拼貼牆「換排列方式」按鈕 ② 開卡進場動畫與特效
> 本輪可自行安排與第四輪(R4)手機/平板/桌機驗證的先後順序,不強制打斷 R4 的驗證流程,
> 但兩輪最終都必須完成並個別驗證通過。

## 1. 留言拼貼牆「換排列方式」按鈕

### 1.1 背景與決策原因
原本評估過「留言自由拖曳定位」,但因為手機/平板/桌機三種版面配置本質上不同,
自由拖曳的座標無法在不同裝置間一致呈現,判斷風險過高,故**不做自由拖曳**。

改為提供「切換排列方式」的按鈕,讓建立者可以在幾種**系統預先設計好、
每種都各自能在三種裝置寬度下正常顯示**的排列邏輯之間切換,解決「這個排法
不好看,想換一種」的實際需求,同時完全不影響 RWD 穩定性。

### 1.2 功能規劃
- 在管理模式畫面新增一個「換個排列方式」按鈕(或提供 3 ~ 4 個排列縮圖供選擇,
  兩種呈現方式擇一,由 Claude Code 判斷哪種操作體驗較好)
- 提供 **3 ~ 4 種**排列邏輯供選擇,方向建議如下(可依實作彈性調整,不要求完全一致):
  1. **緊密拼貼版**:卡片間距縮小、輕微重疊,密度感較高(比照第二輪視覺重新
     設計的預設呈現)
  2. **鬆散錯落版**:卡片間距加大、重疊減少,較為疏朗好閱讀
  3. **依時間整齊排列版**:依留言時間先後,較規律的網格排列,清爽簡潔
  4.（選配）**依有無照片分區版**:有照片的留言與純文字留言分開群組呈現
- 選定的排列方式需儲存在資料庫,下次開啟卡片時維持上次選擇的排列方式,
  不是每次重新整理都跳回預設值

### 1.3 資料庫異動

```sql
ALTER TABLE ecard_cards
ADD COLUMN layout_style TEXT DEFAULT 'collage';
-- 可能值: collage(緊密拼貼) / loose(鬆散錯落) / timeline(時間整齊排列) /
--         grouped(依照片分區，若有實作)
```

修改此欄位一樣走既有的 `edit_card_settings` 函式(第四輪規格已建立),
在原本的參數列表中新增 `p_layout_style TEXT` 一併更新即可,不需要另外
新增函式。

### 1.4 驗證要求
每一種排列方式都必須在手機/平板/桌機三種寬度下實際確認排版正常、
無跑版、無元素重疊或遮擋文字。

## 2. 開卡進場動畫與特效

### 2.1 功能規劃
訪客(或建立者)第一次載入卡片、看到封面畫面時,加入進場動畫效果:

- **基礎動畫**:封面內容(照片/插畫、姓名文字、祝福語)以淡入 + 輕微放大
  的方式出現,取代目前直接顯示的生硬呈現,動畫時間建議 0.6 ~ 1 秒左右,
  不宜過長造成等待感
- **依類別搭配的飄浮特效**(選配裝飾,不影響核心內容):
  - 生日類別:輕巧的紙花或氣球造型元素緩緩飄落
  - 紀念日類別:花瓣或光點緩緩飄落
  - 祝福類別:星星或羽毛造型元素緩緩飄落
  - 感謝類別:葉片或暖色光點緩緩飄落
- 特效元素需與所選質感風格(紙張拼貼／水彩渲染／插畫手繪)的視覺語言一致,
  不要另外引入風格不符的素材(例如插畫手繪風不要用寫實照片級的特效圖片)
- 特效播放一次即可(例如 2 ~ 3 秒後自然消失),不要無限循環播放,
  避免干擾使用者閱讀留言內容,也避免消耗過多裝置效能

### 2.2 效能與相容性考量
- 特效實作方式建議以 CSS 動畫或輕量級 SVG/Canvas 為主,避免引入過重的
  動畫函式庫增加載入時間
- 需在中低階手機瀏覽器測試（或至少確保動畫程式碼有效能節流機制），
  避免動畫造成頁面卡頓
