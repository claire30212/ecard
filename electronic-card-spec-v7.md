# 電子卡片專案 — 第七輪修改規格書

> 此文件為前六份規格書(v1 ~ v6)的補充修正,請一併參考。
> 本輪處理三件事:① 貼紙選單 UI 縮小為網格縮圖 ② 貼紙開放自由選色
> ③ 貼紙風格擴充(新增可愛插畫風、膠帶造型貼紙)

---

## 1. 貼紙選單 UI 調整

實測發現,目前底部貼紙選單是整排大卡片式呈現(每個貼紙佔據畫面大部分寬度,
需橫向滑動才能看到其他貼紙),操作起來不夠一目瞭然。

**修改方向**:改為**小縮圖網格**排列,比照常見表情符號/貼圖鍵盤的呈現方式:

- 手機寬度:建議一排 4 ~ 5 個小縮圖
- 平板/桌機寬度:可以一排顯示更多(例如 6 ~ 8 個),讓使用者一次看到更多選項
- 縮圖本身維持第六輪已調整過的 48px 觸控區域大小,但整體排版改為網格,
  不要單一貼紙佔滿整行寬度
- 若貼紙數量隨本輪擴充變多,選單需要可以捲動(垂直捲動網格,或水平捲動皆可,
  由 Claude Code 依實際版面判斷較順手的方式)

---

## 2. 貼紙開放自由選色

貼紙圖案改為線條可自訂顏色(而非目前固定的單一咖啡色),使用者可以自由
選擇任何顏色。

### 2.1 操作方式
- 點選貼紙後、或貼紙已經放到畫面上時,提供一個顏色選擇入口(例如貼紙旁
  出現一個小色塊圖示,點擊後跳出顏色選擇器/色票)
- 顏色選擇器使用瀏覽器原生的顏色輸入元件即可(`<input type="color">`),
  不需要另外開發複雜的自訂色票 UI,操作單純、瀏覽器相容性好
- 貼紙圖案本身以線條(SVG stroke)為主,顏色套用在線條顏色上

### 2.2 資料庫異動

```sql
-- decorations 欄位（第四輪已建立）的 JSON 結構新增 color 屬性，
-- 不需要異動資料表結構，僅調整前端存入/讀取的 JSON 內容格式：
-- [{"sticker_id": "heart_01", "x_percent": 32.5, "y_percent": 58.0,
--   "rotation": -8, "scale": 1.0, "color": "#B08B85"}, ...]

ALTER TABLE ecard_messages
ADD COLUMN sticker_color TEXT;
-- 訪客留言選的小貼紙顏色，留空則使用預設顏色
```

## 3. 貼紙風格擴充

### 3.1 新增風格方向
除了現有的簡約線條貼紙,新增以下兩種風格,豐富整體選擇:

- **可愛插畫風貼紙**:線條更圓潤、帶一點手繪筆觸感的插畫風格圖案
  (例如笑臉、可愛動物造型、雲朵、小花等),整體感覺更活潑討喜
- **膠帶造型貼紙**:模擬真實紙膠帶的長條矩形造型,可以是素色、
  格紋、圓點、鋸齒邊緣等常見膠帶花紋,強化「手作卡片」的貼紙質感

### 3.2 數量與分類
- 兩種新風格各抓 8 ~ 10 款左右即可,不需要跟原本簡約線條風格數量一樣多
- 貼紙選單建議依風格分類(例如上方增加「簡約」「插畫」「膠帶」的分類頁籤
  或篩選按鈕),避免全部貼紙混在一起，使用者不容易找到想要的類型
- 膠帶造型貼紙因為形狀是長條矩形，貼上後預設角度可以有輕微傾斜（比照
  真實膠帶貼上去的隨性感），使用者仍可再自行旋轉調整
