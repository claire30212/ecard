## 1.【重要,優先處理】貼紙刪除失效 + 正式站功能落差

### 1.1 問題現象
實測發現兩個環境呈現不一致的狀況:

- **正式站(claire30212.github.io/ecard)**:貼紙**無法縮放大小、無法跨頁
  (封面/留言牆之間)拖曳、點擊貼紙右上角的紅色 X 刪除按鈕沒有反應**,
  這幾項都是第八輪(R8)規格書明確要求並回報已完成的功能
- **本機測試環境(localhost:5173)**:縮放功能可以正常運作(看得到縮放手把),
  但**同樣點擊紅色 X 刪除按鈕沒有反應**

### 1.2 請優先確認
1. **正式站是否真的部署了第八輪的最新程式碼**——請確認 GitHub Pages 上
   實際運行的版本、對應的 commit hash,跟本機/repo 裡最新的程式碼是否一致,
   排除「部署卡在舊版本」或「build 快取沒更新」的可能性
2. **貼紙刪除(X 按鈕)在任何環境都點不動**,這是本輪要抓的核心 bug,
   請檢查刪除按鈕的點擊事件是否正確綁定、有沒有被其他疊層(例如縮放手把、
   拖曳事件監聽)攔截掉點擊,導致 X 按鈕形同虛設

### 1.3 驗證要求
修復後,請同時在正式站與本機測試環境都各自驗證一次:貼紙可以縮放、
可以跨頁拖曳(封面拖到留言牆、留言牆拖回封面雙向都要測)、點擊 X
可以正確刪除,三項功能在兩個環境都要一致可用。

## 2. 建卡精靈「下一步」按鈕視覺狀態 bug

### 2.1 問題現象
建立卡片精靈第一步選擇類別時,選擇「祝福」(綠色系)類別後,右下角
「下一步」按鈕**沒有變成啟用狀態該有的顏色**,看起來像是停用(disabled)
的樣子,但實際上仍然可以點擊、正常進入下一步。選擇其他類別(生日/紀念日/
感謝)時,按鈕顏色變化正常,只有「祝福」這個類別會有這個視覺落差。

### 2.2 修復要求
這是視覺呈現與實際功能不一致的問題,容易讓使用者誤以為按鈕故障、
不敢點擊。請檢查按鈕啟用狀態的顏色邏輯是否有跟類別顏色主題掛勾的地方
（例如用該類別的強調色當按鈕背景色，导致祝福類別的綠色系強調色跟
按鈕預設底色太接近而看不出對比），確保四個類別選擇後,「下一步」按鈕
都能清楚呈現「可以點擊」的視覺狀態。

## 4. 貼紙新增旋轉功能

部分貼紙貼上去的角度不理想（例如膠帶貼紙的預設隨機傾斜角度不符合使用者
想要的方向感），需要能手動調整貼紙的旋轉角度。

### 4.1 操作方式
比照第八輪已實作的縮放手把模式,在貼紙上新增一個**旋轉手把**
(常見做法是貼紙上方伸出一個小圓點,拖曳該點沿圓周方向轉動即可調整角度),
與縮放手把分開、不要互相干擾操作。

### 4.2 技術要求
- 旋轉角度沿用既有 `decorations` JSON 結構裡的 `rotation` 欄位
  (第四輪規格書已定義),不需要新增資料庫欄位
- 手機觸控情境下,旋轉手把的可觸控範圍一樣需要足夠大,避免不好精準操作

## 5. 貼紙新增透明度調整功能

### 5.1 需求
希望貼紙(尤其是膠帶造型貼紙)可以調整透明度,讓貼紙呈現半透明效果,
更貼近真實紙膠帶貼上去、底下圖案隱約透出來的質感。

### 5.2 建議做法
建議將透明度調整功能**套用在所有貼紙類型**(不限膠帶),做法較單純一致,
不需要特別為膠帶類別另外寫一套邏輯:

- 選定貼紙後,提供一個透明度滑桿(例如範圍 30% ~ 100%),即選即生效
- 透明度數值存入 `decorations` JSON 結構,新增 `opacity` 屬性即可
  (格式範例:`{"sticker_id": "tape_01", ..., "opacity": 0.7}`),
  不需要異動資料庫欄位結構
